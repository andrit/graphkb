System architecture

The five-layer pipeline from content ingestion through to the user interface. Each layer is a set of composable pure functions with typed inputs and outputs. Click any component to learn more.

Project structure

Monorepo layout with pnpm workspaces and Turborepo. TypeScript packages on the left, Python services on the right, connected via a Redis/BullMQ message queue. Click any package for details.

Docker Compose infrastructure

Three containerized services provide the data layer. Application code runs on the host for fast iteration. Named volumes persist data across restarts. Click any service for configuration details.

Graph data model (ontology)

Two layers: structural nodes (coral, system-created from ingestion) and semantic nodes (purple, discovered by NLP or curated by you). User-created nodes in teal. The RELATED_TO edge between entities is the rhizomatic connection that surfaces unexpected relationships. Click any node for details.

Concrete example: PDF ingestion

What happens when you upload "Why Functional Programming Matters" by John Hughes. The structural layer (coral) is created by the pipeline. The semantic layer (purple) emerges from NLP extraction. The rhizomatic connection appears when a shared Entity links this paper to a previously ingested React tutorial.
